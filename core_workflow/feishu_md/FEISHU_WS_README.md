# 飞书长连接使用说明

本项目支持使用飞书长连接（WebSocket）接收事件消息，替代传统的 Webhook 方式。

## 优势

- **开发方便快捷**：无需公网 IP 或域名，无需内网穿透工具
- **加密传输**：内置通信加密和鉴权，无需额外解密、验签逻辑
- **本地开发友好**：在本地开发环境中即可接收事件消息

## 使用方式

### 方式一：集成到 FastAPI 应用（推荐）

启动 FastAPI 应用时会自动启动飞书长连接客户端：

```bash
# 1. 安装依赖
pip install lark-oapi

# 2. 配置环境变量
export FEISHU_APP_ID=cli_xxxxxx
export FEISHU_APP_SECRET=xxxxxxxx

# 3. 启动应用
python -m uvicorn app.main:app --reload
```

### 方式二：独立运行长连接服务端

如果只需要飞书长连接功能，可以单独运行：

```bash
# 1. 配置环境变量
export FEISHU_APP_ID=cli_xxxxxx
export FEISHU_APP_SECRET=xxxxxxxx

# 2. 启动长连接服务端
python feishu_ws_server.py
```

## 飞书开发者后台配置

1. 登录 [飞书开发者后台](https://open.feishu.cn/app)，选择企业自建应用
   - 注意：长连接模式仅支持企业自建应用，不支持商店应用

2. 进入 **事件与回调 > 事件配置** 页面

3. 编辑订阅方式，选择 **使用长连接接收事件**，并保存
   - ⚠️ 必须确保本地客户端启动正常，有长连接在线的情况下，才能保存成功

4. 添加需要订阅的事件，例如：
   - `im.message.receive_v1` - 接收消息事件

## 环境变量

| 变量名 | 说明 | 获取方式 |
|--------|------|----------|
| `FEISHU_APP_ID` | 飞书应用 ID | 开发者后台 > 凭证与基础信息 > 应用凭证 |
| `FEISHU_APP_SECRET` | 飞书应用密钥 | 开发者后台 > 凭证与基础信息 > 应用凭证 |

## 注意事项

1. **超时处理**：长连接模式下接收到消息后，需要在 3 秒内处理完成且不抛出异常，否则会触发超时重推机制

2. **连接限制**：每个应用最多建立 50 个连接

3. **集群模式**：长连接模式的消息推送为集群模式，不支持广播。如果同一应用部署了多个客户端，只有随机一个客户端会收到消息

4. **网络要求**：服务器需要能够访问公网

## 代码结构

```
app/service/
├── feishu_ws_client.py      # 基础长连接客户端（阻塞模式）
└── feishu_ws_manager.py     # 长连接管理器（后台线程模式）

feishu_ws_server.py           # 独立运行长连接服务端
```

## 处理的消息类型

目前支持处理 `im.message.receive_v1` 事件（接收消息 v2.0），只处理文本消息类型。

收到消息后，会：
1. 检查用户是否有进行中的工作流
2. 如果有，将消息提交到该工作流
3. 如果没有，创建新工作流

## 常见问题

### 保存订阅方式时提示"没有可用的长连接"

确保长连接客户端已启动并正常运行，可以在控制台看到 `connected to wss://xxxxx` 的日志后再保存。

### 消息重复接收

如果处理消息超过 3 秒，飞书会触发超时重推机制。请确保消息处理逻辑在 3 秒内完成，或者使用异步方式处理。

### 多个客户端同时运行

由于是集群模式，同一应用部署多个客户端时，只有随机一个会收到消息。如需广播，请考虑其他方案。
